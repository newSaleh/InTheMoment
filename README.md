# InTheMoment 🌿⏳

بوت تيليقرام بسيط، مهمّته الوحيدة أن يرسل لك بين الحين والآخر تذكيرًا
بأن تعيش هذه اللحظة، وبأننا جميعًا فانون. تذكيرات عشوائية (2-3 يوميًا)
في أوقات غير متوقعة، حتى لا تتحول إلى روتين تعتاد عليه وتتجاهله.

يوجد طريقتان لتشغيله:

1. **بوت حي دائم الاتصال** (`bot/main.py`) — يتطلب خادمًا يعمل باستمرار
   (محليًا، Docker، أو systemd)، لكن ردوده فورية.
2. **مجانًا بالكامل عبر GitHub Actions** (`bot/serverless.py`) — لا يحتاج
   أي خادم، يعمل داخل هذا المستودع نفسه عبر فحص دوري كل 15 دقيقة تقريبًا،
   لكن ردوده على الأوامر قد تتأخر بضع دقائق بدل أن تكون فورية.

القسم التالي "كيف يعمل" يصف السلوك المشترك بين الطريقتين. اذهب مباشرة
لقسم [التشغيل المجاني عبر GitHub Actions](#التشغيل-المجاني-عبر-github-actions-بدون-خادم)
إن كنت تريد الخيار الذي لا يحتاج خادمًا.

## كيف يعمل

- عند الاشتراك (`/start`)، ينضم المستخدم إلى قائمة المشتركين (تُحفظ في
  قاعدة بيانات SQLite محلية تبقى بعد إعادة تشغيل البوت).
- كل يوم عند الساعة 00:05 (بتوقيت `TZ_NAME`)، يختار البوت لكل مشترك
  عددًا عشوائيًا من التذكيرات (2 أو 3 افتراضيًا) في أوقات عشوائية موزّعة
  ضمن نافذة زمنية نهارية (9 صباحًا - 11 مساءً افتراضيًا)، بفاصل زمني لا
  يقل عن ساعة بين تذكير وآخر.
- عند إرسال كل تذكير، يختار البوت رسالة عشوائية من بنك رسائل (`bot/messages.py`)
  يمزج بين تذكيرات "تذكّر الفناء" (memento mori) وتذكيرات "الحضور في اللحظة"
  (mindfulness)، مع تجنّب تكرار نفس الرسالة مرتين متتاليتين لنفس الشخص.
- الاشتراك الجديد في منتصف اليوم يحصل فورًا على جدول تذكيرات للوقت
  المتبقي من نفس اليوم، فلا داعي للانتظار حتى الغد.

## الأوامر

| الأمر | الوظيفة |
|---|---|
| `/start` | الاشتراك في التذكيرات |
| `/now` | إرسال تذكير عشوائي فورًا (بدون التأثير على الجدول اليومي) |
| `/stop` | إلغاء الاشتراك وإيقاف كل التذكيرات القادمة |
| `/help` | عرض شرح مختصر عن البوت |

## التشغيل محليًا

يتطلب Python 3.11+.

```bash
python -m venv venv
source venv/bin/activate
pip install -r requirements.txt

cp .env.example .env
# افتح .env وضع فيه BOT_TOKEN الخاص بك (احصل عليه من @BotFather على تيليقرام)

python -m bot.main
```

## التشغيل عبر Docker

```bash
cp .env.example .env
# عدّل .env كما تريد
docker compose up -d --build
```

قاعدة البيانات تُحفظ في مجلد `./data` على الجهاز المضيف حتى تبقى بعد
إعادة بناء الحاوية.

## التشغيل كخدمة systemd (على خادم Linux)

1. انسخ المشروع إلى `/opt/inthemoment` وأنشئ بيئة افتراضية فيه (`venv`)
   ثم ثبّت المتطلبات، وأنشئ ملف `.env` بجانبها.
2. انسخ `deploy/inthemoment-bot.service` إلى `/etc/systemd/system/`.
3. فعّل الخدمة:

```bash
sudo systemctl daemon-reload
sudo systemctl enable --now inthemoment-bot
```

## التشغيل المجاني عبر GitHub Actions (بدون خادم)

هذه الطريقة تستخدم `bot/serverless.py` بدل `bot/main.py`: سكربت خفيف
يعمل مرة واحدة في كل تشغيل، يفحص رسائل تيليقرام الجديدة، يرسل أي
تذكيرات مستحقة، ثم يحفظ حالته في `data/state.json` **داخل المستودع
نفسه** — تشغيل GitHub Actions المُجدوَل في `.github/workflows/reminders.yml`
هو من يستدعيه كل 15 دقيقة تقريبًا ويحفظ التغييرات تلقائيًا. لا حاجة
لأي خادم أو حساب خارجي.

### الإعداد (مرة واحدة فقط)

1. **أضف توكن البوت كسر (Secret)**، وليس في أي ملف عادي:
   - اذهب إلى صفحة المستودع على GitHub → **Settings** → **Secrets and
     variables** → **Actions** → تبويب **Secrets** → **New repository secret**.
   - الاسم: `BOT_TOKEN` — القيمة: التوكن الذي أعطاك إياه BotFather.
2. **تأكد أن الصلاحيات تسمح بالكتابة** (لكي يتمكن الـ workflow من حفظ
   `data/state.json` تلقائيًا بعد كل تشغيل):
   - **Settings** → **Actions** → **General** → قسم **Workflow permissions**
     → اختر **Read and write permissions** → احفظ.
3. *(اختياري)* لتخصيص التوقيت بدل القيم الافتراضية، أضف من نفس صفحة
   **Secrets and variables → Actions** لكن من تبويب **Variables** متغيرات
   مثل `TZ_NAME`، `ACTIVE_HOUR_START`، `ACTIVE_HOUR_END`،
   `MIN_DAILY_REMINDERS`، `MAX_DAILY_REMINDERS`، `MIN_GAP_MINUTES`
   (نفس الأسماء الموجودة في `.env.example`). إن لم تُضِفها، تُستخدم
   القيم الافتراضية المعقولة تلقائيًا.

### التجربة

- اذهب إلى تبويب **Actions** في المستودع → اختر workflow باسم
  **Send reminders** → اضغط **Run workflow** لتشغيله فورًا يدويًا بدل
  انتظار الجدول الزمني (مفيد للاختبار الأول).
- بعد انتهاء التشغيل (يستغرق ثوانٍ)، افتح تيليقرام وأرسل `/start` للبوت،
  ثم شغّل الـ workflow يدويًا مرة أخرى (أو انتظر حتى 15 دقيقة) لترى الرد.
- جرّب `/now` لتلقي تذكير فوري (بعد أن يعمل التشغيل التالي)، و`/stop`
  لإلغاء الاشتراك.

### ملاحظات مهمة

- بما أن هذا يعمل بفحص دوري كل 15 دقيقة تقريبًا (وليس اتصالًا حيًا
  دائمًا)، فردود الأوامر ليست فورية — قد تأخذ من دقائق إلى نحو ربع
  ساعة حسب توقيت آخر تشغيل، وأحيانًا أطول قليلًا إذا كان GitHub مزدحمًا.
  هذا طبيعي وهو الثمن مقابل أن يكون التشغيل مجانيًا بالكامل.
- بما أن هذا المستودع عام (public)، فتشغيل GitHub Actions مجاني بلا حدود
  عملية تقريبًا. لو حوّلت المستودع إلى خاص (private) مستقبلًا، ستخضع
  لحصة شهرية محدودة من دقائق التشغيل.
- لا تشغّل `bot/main.py` (الطريقة الحيّة) و`bot/serverless.py` (طريقة
  GitHub Actions) في نفس الوقت لنفس التوكن — سيتنازعان على استقبال
  الرسائل من تيليقرام.

## التخصيص

كل إعدادات التوقيت موجودة في `.env` (انظر `.env.example` للتفاصيل):
عدد التذكيرات اليومية، النافذة الزمنية المسموح بها، الحد الأدنى للفاصل
بين تذكيرين، والمنطقة الزمنية.

### إضافة أو تعديل أو حذف الاقتباسات

كل نصوص التذكيرات موجودة في ملف `bot/quotes.txt` — **ملف نصي عادي، لا
كود برمجي**. كل اقتباس هو كتلة نص مستقلة، وتُفصل الاقتباسات عن بعضها
بسطر يحتوي على `---` فقط. لتعديل المحتوى:

- **لإضافة اقتباس جديد**: أضف نص الاقتباس في أي مكان بالملف، واجعل قبله
  وبعده سطر `---` يفصله عمّا حوله.
- **لتعديل اقتباس موجود**: عدّل نصّه مباشرة داخل كتلته.
- **لحذف اقتباس**: احذف كتلته بالكامل (نصّه وأحد سطرَي `---` المحيطين
  به حتى لا يتبقى فاصل زائد).

لا حاجة لعلامات اقتباسٍ أو تنصيصٍ خاص بلغة بايثون؛ الملف نص عادي بالكامل
ويدعم الأسطر المتعددة داخل نفس الاقتباس بلا أي مشاكل. يقرأ البوت هذا
الملف عند بدء التشغيل، فأي تعديل يتطلب إعادة تشغيل البوت (تلقائيًا في
حالة GitHub Actions، أو يدويًا إذا كنت تُشغّل `bot/main.py` كعملية
دائمة) ليظهر أثره.

بإمكانك أيضًا خلط اقتباسات من مصادر مختلفة في نفس الملف؛ فقط اجعل كل
كتلة تحمل نصّها الكامل (بما في ذلك أي إسناد للمصدر إن رغبت) وافصلها عن
جاراتها بسطر `---`.

## ملاحظة

المحتوى الحالي هو اقتباسات من كتاب «التأملات» لماركوس أوريليوس، تدور
حول تذكّر الفناء (memento mori) والعيش وفق العقل والفضيلة. إن شعرت في
أي وقت أن التذكيرات تزعجك أكثر مما تفيدك، يمكنك ببساطة إرسال `/stop`.
